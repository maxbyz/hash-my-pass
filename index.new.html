Hash My Pass is a Javascript bookmarklet that generates passwords guaranteed to be unique for each site so you'll never have to reuse the same password for different sites, which, as everyone knows, is a security risk.

The great thing is, you only have to memorize a single master password, and this is NEVER stored anywhere. At the same time, you're without the security problems of sharing passwords across sites.

password reuse is unsafe. but remembering different passwords is inconvenient

The main concept is based on hashing a single master password with the current site's domain.

A hash is not encryption – it cannot be decrypted back to the original text (it is a "one-way" cryptographic function). Therefore, it is impossible to obtain the master password from the generated site-specific password, so even if your acount on one website is compromised, all the other accounts are perfectly safe.

Since the hash is always the same given the same inputs, it can be generated on-the-fly, so there's no need to store anything: none of the generated passwords are stored anywhere.

As a javascript bookmarklet, it works on the client side, which means that the master password is never sent through the network.

Hash My Pass was started as a team project for Codebits 2011. The team members Waldir Pimenta, Abel Soares, Sérgio Laranjeira and Miguel Oliveira had to complete the project from scratch in 48 hours for the Codebits competition.

The original idea was inspired in Nic Wolff's Password generator. Hash My Pass improves upon Nic's generator in the following ways:

  • It is hosted on a publicly available version-control platform (github), and can easily be improved by others, be forked, have pages translated, issues reported, etc.
  • It attempts to cater to the different password requirements (length, allow/forbid alphanum/symbols, etc.) that each site implements.
  • It uses a less cheap hack than appending "1a" to the generated password to make sure that at least a letter and a number is included (and only does so when relevant to the particular site)
  • The bookmarklet prompt doesn't mask the pass. That isn't possible using Javascript's native popup boxes. We solved this by using a lightbox-type prompt emulator that uses an <input type="password"> to provide the masking using HTML's native mechanisms
  • The domain recognition code uses a more powerful regex in order to cover edge cases (country-specific TLDs, for instance).

What if someone finds my master password?
Well, what if they found the current password you use in most websites? Of course Hash My Pass is not 100% secure (nothing is), but it’s still safer than reusing passwords.
Does this also save my username?
No. it is very rare to be able to keep your username across websites, so this wouldn’t be very useful. besides, most browsers may save username/password combinations (Of course, make sure to not let other people use your browser profile).
